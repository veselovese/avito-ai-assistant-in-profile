Есть два варианта: я вижу комментарий в коде сервера "Настройка CORS" и могу уснановить нужные header\`ы там или могу описать proxy на стороне vite. На этапе разработки буду использовать proxy, свое решение на back`е прилагаю.
```
fastify.use((_, reply, next) => {
  reply.setHeader('Access-Control-Allow-Origin', '*');
  reply.setHeader('Access-Control-Allow-Methods', 'GET,POST,PUT,OPTIONS')
  reply.setHeader('Access-Control-Allow-Headers', 'Content-Type')

  if (_.method === 'OPTIONS') {
    reply.statusCode = 204
    reply.end()
    return
  }
  next();
});
```

- 25.03 20:30 Только сейчас заметил эту особенность back\`а: он не принимает необязательные поля в виде "brand": "". Он требует, чтобы пустые поля вообще не отправлялись. Дабавил на фронт специальную функцию-клинер для отправляемого тела запроса.
```
const cleanParams = (params: Record<string, any>) => {
        return Object.fromEntries(
            Object.entries(params).filter(([_, value]) => {
                return value !== '' && value !== null && value !== undefined;
            })
        );
    };  
```
